# String Machine Path Index

Date: 2026-07-04

Scope: cleanup index for paths that still use strings as active generation, routing, parsing, continuation, dedupe, or success logic. This is not a list of every string in the repo. Display-only labels and formula echoes are included only when they are reused as input, fallback logic, identity, dedupe, or generation state.

Current implementation target: Andrews Logic Authority.

Rule of thumb: the engine should produce two related records. #1 is the formula, backed by formula slots and route contracts. #2 is that same formula realized with no formula characters, only letters, backed by segment/result frames where the caller can prove those frames are complete; compatibility surface fields are still indexed until their callers are fully segment-derived. `formulaEcho`, `result`, `surface`, slash-joined text, DOM dataset strings, and scrubbed formula text should be terminal display artifacts, not active grammar logic.
